# Atlas MVP 交付清单（QA 落地版）

> 更新日期：2026-03-11  
> 维护角色：QA / 交付协同  
> 目的：把当前 Atlas MVP 从“文档规划”收敛到“可验证交付”。

---

## 1. MVP 目标边界

本清单以当前仓库现状为准，优先确保以下最小闭环：

1. 能启动前后端基础工程
2. 能完成 mock 登录
3. 员工可查看个人班表
4. 店长可创建/查看排班草稿并做基础校验
5. 命中特殊规则后可进入审批页查看与处理
6. 文档、接口、代码骨架三者口径一致

> 注意：当前仓库尚未具备真实企微登录、真实数据库、真实审批流实现，因此本轮交付默认先以 mock 闭环为主，真实集成作为后续迭代项。

## 2. MVP 功能清单

### 2.1 P0 必须交付

#### A. 工程与运行
- [x] 前端工程目录存在（`atlas-web/`）
- [ ] 前端可执行 `npm install`
